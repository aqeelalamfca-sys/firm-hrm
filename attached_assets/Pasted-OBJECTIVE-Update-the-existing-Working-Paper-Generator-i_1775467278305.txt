NEW TARGET WORKFLOW
Make the working paper generator run in this order:

STEP 1 — TEMPLATE-BASED UPLOAD
User uploads only the one-sheet Excel template as the primary input.
Optional supporting files may also be uploaded later, but the one-sheet template is enough to run the core process.

STEP 2 — TEMPLATE PARSING

- entity name
- company type

- line id

- debit transaction value
- credit transaction value
- normal balance
- wp area
- risk level
- procedure scale
- ai gl flag
- gl generation priority
- remarks

STEP 3 — VALIDATION ENGINE
Before further processing, validate:
- required header fields exist
- the sheet name is accepted even if changed slightly
- account codes are not blank for material rows
- Current_Year values are numeric where needed
- debit / credit transaction values are numeric
- statement types are only BS / PL / OCI / EQ / CF if permitted
- normal balance values are valid
- duplicate line/account conflicts are flagged
- missing major heads / FS sections are flagged
- blank rows are ignored safely
- user sees exact validation errors in frontend

STEP 4 — DATA NORMALIZATION
Convert the one-sheet template into normalized backend datasets:
- session master
- entity profile
- fs line items
- account master / coa session table
- TB seed lines
- GL generation queue
- variable seed values
- risk seed values
- working paper trigger seed values

STEP 5 — AUTO VARIABLES
Auto-fill all possible variables from the template without waiting for manual entry.
Map top-sheet fields and financial rows into existing variable engine.
If a variable cannot be derived, mark it as:
- missing
- review required
- unable to infer
Do not leave silent blanks.

STEP 6 — TRIAL BALANCE GENERATION
Generate a balanced TB from the one-sheet data.
Rules:
- each line maps into a TB account using account code + account name
- CY closing shall align with uploaded current year figure
- PY shall align with prior year figure where applicable
- debit / credit side shall respect normal balance
- total debits and total credits must reconcile
- if balancing difference exists, log exception and show exact reason
- store the generated TB in session tables and expose download/export

STEP 7 — GENERAL LEDGER GENERATION
Generate transaction-wise GL for rows where AI_GL_Flag = Yes or priority requires generation.
Rules:
- opening balance + transactions = closing balance
- transaction count should be logical based on amount, risk level, and procedure scale
- large or high-risk balances should generate richer transaction detail
- debit and credit transaction value columns should guide the volume and direction of generated postings
- keep the GL realistic, not random nonsense
- ensure each GL account ties back to TB and FS values exactly or within controlled zero-difference logic
- static balances with no movement may generate carry-forward logic
- store account summary and entries separately in DB

STEP 8 — AUDIT ENGINE LINKAGE
Push the parsed and normalized data into:
- wp triggers
- assertions
- sampling rules
- analytics engine
- control matrix seeds
- evidence requirements
- head readiness status
- exceptions panel
The uploaded WP_Area, Risk_Level, Procedure_Scale, AI_GL_Flag, and GL_Generation_Priority must directly influence working paper depth.

STEP 9 — HEADWISE WORKING PAPER GENERATION
Generate outputs in proper sequence:
1. Trial Balance
2. General Ledger
3. Pre-Planning
4. Trial Balance & GL
5. Client Documents
6. OB Verification
7. Planning
8. Execution
9. Finalization
10. Deliverables
11. EQCR
12. Inspection

Each head must:
- only start after prior dependency is complete
- show status clearly
- save generated content in DB/file records
- support approval and export
- use session data from the uploaded template
- include risk-responsive and balance-responsive narrative

STEP 10 — EXPORT
Allow export of:
- TB in Excel
- GL in Excel
- Working papers in Word / Excel / PDF as per head
- bundled export
All exports must use actual session data, not placeholder text.
