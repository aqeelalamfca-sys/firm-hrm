### 4. Application Features

The web application must have:

**Input Section:**
- Checkbox grid for all 14 company types (as listed above)
- "Select All" and "Clear All" buttons
- Real-time update on selection change

**Output Section - Two Views:**

**View 1: Summary Dashboard**
- Total required working papers count
- Breakdown by phase (Pre-Engagement, Planning, Risk Assessment, etc.)
- Missing papers count (by company type selection)

**View 2: Detailed Table**
- Sortable/filterable table with columns:
  - ID
  - Phase
  - Working Paper Name
  - Status (Required / Optional / Not Applicable)
  - Applicable To (human-readable list of company types)

**View 3: Export Functionality**
- Button to export selected working papers to CSV
- Button to print as checklist

### 5. User Interface Requirements

- Clean, professional design (suitable for audit firms)
- Responsive (works on desktop and tablet)
- Dark/light mode toggle (nice to have)
- Progress indicator showing completion percentage
- Search/filter by working paper name or phase

### 6. Logic Rules

- If user selects NO company types → show "Please select at least one company type"
- If a working paper's `applicableTo` matches ANY selected company type → Status = "Required"
- If a working paper is marked as "Required for ALL" → Status = "Required" regardless of selection
- If a working paper has no match but is not universal → Status = "Not Applicable"
- Some working papers have `optionalFor` - show as "Optional" if only optional conditions met

### 7. Data Persistence (Bonus)

- Save user's company type selection in localStorage
- Load saved selection on page refresh

- Use vanilla HTML/CSS/JS (no external frameworks required, but Tailwind CSS is acceptable)

- Mobile-friendly CSS grid/flexbox
- No console errors
